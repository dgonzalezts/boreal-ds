The `bds-text-field` component is a foundational element for capturing user input within forms. It extends the standard HTML `<input>` element with enhanced features, including integrated labels, helper text, validation states, character counting, and password visibility toggle, all while ensuring full accessibility and seamless form integration.

<Callout variant="info" icon="ℹ️">
  This component integrates with HTML forms via the `name` attribute for native form participation. For more details, see the [Form Integration](#form-integration) section.
</Callout>

<Subtitle>How to use it</Subtitle>

1. Import the component into your project.

```ts
import '@telesign/boreal-web-components/bds-text-field';
```

2. Use the `<bds-text-field>` tag to display the component in your HTML.

```html
<bds-text-field
  name="email"
  label="Email address"
  placeholder="you@company.com"
  helper-text="We will never share your email"
  required
  validation-timing="blur"
></bds-text-field>
```

<Subtitle>When to use it</Subtitle>

Use `bds-text-field` when:

- **Forms require text input** — any field that collects a single line of text, such as names, emails, or usernames.
- **Validated fields** — inputs that need built-in or custom validation with configurable timing (blur, input, change, or submit).
- **Password entry** — use `type="password"` to enable the visibility toggle button.
- **Search inputs** — use the `plain` variant and optionally the prefix slot for an inline search icon.
- **Read-only display** — use `readonly` to show a value that cannot be edited but is still submitted with the form.

## Form Integration

The `bds-text-field` is designed for seamless integration with native HTML forms, supporting standard validation attributes and custom validation logic.

<Callout variant="tip" icon="💡">
  For best accessibility and form association, provide a unique `name` for each field. The component generates internal IDs automatically.
</Callout>

### Form Association

The component automatically associates with HTML forms using the `name` attribute.

```html
<form id="user-registration">
  <bds-text-field
    name="username"
    label="Username"
    required
  ></bds-text-field>
  <bds-text-field
    name="email"
    label="Email"
    type="text"
    required
  ></bds-text-field>
</form>
```

<Subtitle>Accessibility</Subtitle>

The TextField component is designed with accessibility in mind.

- **ARIA attributes**: The inner `<input>` is wired with `aria-labelledby` (points to the label element), `aria-describedby` (points to the helper or error text), `aria-invalid="true"` when `error` is set, and `aria-required="true"` when `required` is set.
- **Keyboard navigation**: `Tab` to focus the field, type to input text. The clear button and password visibility toggle are reachable by keyboard when visible.
- **Screen reader announcements**: The label, sublabel, helper text, and error message are all announced by assistive technology. Error state changes are communicated via `aria-invalid`.
- **Best practices**:
  - Always provide a descriptive `label`.
  - Use `helper-text` for supplementary instructions.
  - Ensure custom `error-message` values are clear and concise.
